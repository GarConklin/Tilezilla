Tilezilla Guest User System Specification
Overview

Tilezilla supports anonymous guest play for Daily Challenges only.

Guest users are automatically assigned a unique Guest Code on first visit. This code is used for analytics, logging, troubleshooting, and tracking daily challenge participation.

Guest users are intentionally restricted from most game features to encourage account creation.

Guest User Creation
Trigger

On site load:

if (!localStorage.getItem('guest_code') && !userLoggedIn) {
    createGuestUser();
}
Guest Code Format
Guest-TZ-A####-XX

Examples:

Guest-TZ-A4821-QM
Guest-TZ-A0194-KD
Guest-TZ-A7777-ZX
Components
Part	Description
Guest	User Type
TZ	Tilezilla
A	Anonymous Identifier Prefix
####	Random 4 digit number
XX	Random uppercase letters

Generation Example:

function generateGuestCode() {
    const num = Math.floor(Math.random() * 10000)
        .toString()
        .padStart(4, '0');

    const letters =
        String.fromCharCode(65 + Math.floor(Math.random() * 26)) +
        String.fromCharCode(65 + Math.floor(Math.random() * 26));

    return `Guest-TZ-A${num}-${letters}`;
}
Local Storage

Store:

localStorage.setItem('guest_code', guestCode);

Example:

guest_code = Guest-TZ-A4821-QM

Guest code remains until:

Browser cache cleared
Local storage cleared
User registers and converts account
Database Table
CREATE TABLE guest_users (
    guest_id INT AUTO_INCREMENT PRIMARY KEY,
    guest_code VARCHAR(32) UNIQUE NOT NULL,
    created_at DATETIME NOT NULL,
    last_seen DATETIME NOT NULL,
    total_daily_attempts INT DEFAULT 0,
    total_daily_solves INT DEFAULT 0
);

The guest experience should allow a visitor to:

Immediately play today's Daily Challenge.
Experience the core Tilezilla gameplay.
Generate useful analytics through Guest Codes.
Encounter gentle account-creation prompts.
Be unable to access progression, rewards, hints, statistics, or other long-term systems until registered.
